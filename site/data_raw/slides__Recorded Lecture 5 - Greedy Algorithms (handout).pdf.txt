Greedy Algorithm 
FIT2004: Seminar 5 - Greedy (Graph) Algorithms oGreedy paradigm for algorithms: make a locally optimal  choice at each stage 
and commit to it. 
oOften it is easy to come up with ideas to solve a problem using a greedy 
strategy, but in many cases the proposed greedy algorithm does not find the 
overall optimal solution for the problem. 
oCorrectness proofs are very important . 
oToday we will take a look at three very important greedy algorithms: 
oDijkstra’s algorithm solves the shortest path problem in graphs with non-negative weights. 
oPrim’s and Kruskal’s algorithms present two different ways to find a minimum spanning tree of a 
graph (an important problem for which there are actually multiple distinct greedy strategies that 
work fine).

Dijkstra’s Algorithm 
FIT2004: Seminar 5 - Greedy (Graph) Algorithms 
oAlgorithm for solving the single source, all targets shortest path problem on 
graphs with non-negative weights . Closely related to BFS. 
oIt keeps track of a set S of nodes whose distance to the source has already 
been determined. 
oInitially S only contains the source node, and it grows by one element at a 
time until containing all nodes that are reachable from the source node. 
oAt each iteration, from all nodes that are one edge away from S, add to S the 
node u that has the smallest distance to the source .
oBy doing so, the overall effect is that the nodes will be added to S in 
increasing order of distance to the source (as we will see soon).

FIT2004: Seminar 5 - Greedy (Graph) Algorithms Time Complexity: 
oWhile loop executes θ(V) times 
oFind  the vertex with smallest distance and delete  it: depends on priority queue 
implementation (Q.extract_min) 
oEach edge visited once → θ(E) 
oUpdating the priority queue: depends on implementation (Q.update) 
oRelaxation is θ(1) since we can find distances and compare them in θ(1)
oTotal cost: θ(E*Q.update+ V*Q.extract_min)

Dijkstra’s Algorithm using min-heap 
FIT2004: Seminar 5 - Greedy (Graph) Algorithms Required additional data structure: 
o Create an array called Vertices . 
o Vertices [i] will record the location  of i-th vertex in the min-heap 
Updating the distance of a vertex v in min-heap in θ(log V )
o Find the location in the queue (heap) in θ(1) using Vertices 
o Now do the normal heap-up operation in θ(log V) 
oFor each swap performed between two vertices x and y during the upHeap 
oUpdate Vertices [x] and Vertices [y] to record their updated locations  in the 
min-heap

MST Algorithms: Prim’s Algorithm 
FIT2004: Seminar 5 - Greedy (Graph) Algorithms Prim’s Algorithm (small modification of Dijkstra’s Algorithm) 
oWe start with no edges in T, so there are V connected components in T (each with a single 
node) and select one node as the source. 
oAt each iteration, we add to T the shortest edge that links the connected component 
containing the source to a different connected component .
oIn each iteration, the number of connected components decreases by 1 and the number 
of nodes in the connected component containing the source increases by 1. 
oCycles are avoided as no edges linking two nodes that are already in the same connected 
component are ever added to T. 
oTime complexity: θ(E log V)

MST Algorithms: Kruskal’s Algorithm 
FIT2004: Seminar 5 - Greedy (Graph) Algorithms Kruskal’s Algorithm 
oWe start with no edges in T, so there are V connected components in T (each with a single 
node). 
oWe process edges in ascending order of edge weights, and add an edge e to T if this 
addition does not create a cycle. 
oEach time that one edge e is added to T, e is the smallest edge that links two distinct 
connected components of T. 
oCycles are avoided as no edges linking two nodes that are already in the same connected 
component are ever added to T. At any point of the execution T is a forest (i.e., a set of 
trees)
oTime complexity: θ(E log V).

who is its own parent (i.e. a root). That vertex ID is 
the set ID 
Union(u,v): If find(u) != find(v), set parent[find(u)] = 
find(v) (or vice versa) 
What is the complexity of union and find? 
Union is O(find). Find could in theory be θ(V), but if 
we can keep the heights of the trees low, then it will 
be at most θ(max height)

Concluding Remarks 
FIT2004: Seminar 5 - Greedy (Graph) Algorithms Take home message 
oDijkstra’s algorithm is a greedy algorithm for determining shortest paths on 
graphs with non-negative weights. 
oPrim’s and Kruskal’s algorithms are both greedy algorithms that correctly 
determine minimum spanning trees. While Prim’s algorithm keeps growing 
the same connected component at each iteration; Kruskal’s algorithm 
merges the two closest connected components.
